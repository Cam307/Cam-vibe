# AI章节创作规范文档

> 版本：v2.2
> 创建日期：2026-02-15
> 用途：基于细纲创作章节时的规范和要求
> 包含：对话写作规范、爽点设计规范、钩子设计规范
> 输出模板：详见《小说章节输出模板.md》

---

## 一、使用说明

### 1.1 文档用途
AI在创作每一章时，必须：
1. 基于细纲创作章节内容
2. 应用对话写作规范
3. 应用爽点设计规范
4. 应用钩子设计规范
5. 按照《小说章节输出模板.md》输出

**【字数硬性要求】**  
每章**正文**（不含标点）**必须达到 4000 字以上**。若初稿不足 4000 字，**必须在本章内补写**（增加对话、动作、心理、环境或情节细节等），**直至正文字数（不含标点）≥4000 字后方可输出**。未达标不得视为本章完成。

### 1.2 使用流程
```
1. 细纲已确定（可为全书或当前要写的批次）
  ↓
2. 按「分批生成」执行，不要一次性写完所有章节：
   - 每批生成 3 章；
   - 每批完成后，必须生成「本批剧情摘要」；
   - 下一批创作时，将「上一批剧情摘要」作为上下文提供给 AI，再基于细纲写本批章节。
  ↓
3. AI 执行：基于本规范 + 当前批次细纲 + 上一批剧情摘要（若有）创作本批章节
  ↓
4. AI 输出：按《小说章节输出模板》输出本批各章；本批全部完成后，输出「本批剧情摘要」
  ↓
5. 进入下一步：自我检查（可每章或每批结束后进行）
```

### 1.3 分批生成机制

**为何分批：**  
当前 AI 模型在一次性输出大量章节时，容易出现后半段逻辑崩坏、人设漂移、细节遗忘。采用「每批 3 章 + 剧情摘要传递」可稳定前后一致性与人设。

**执行约定：**
- **每批章数**：固定为 **3 章**（即每轮只创作 3 章正文）。
- **每批结束后**：必须产出「**本批剧情摘要**」，格式见《小说章节输出模板》中的「本批剧情摘要」区块。
- **下一批开始前**：将「**上一批剧情摘要**」作为上下文提供给 AI，再基于细纲写本批 3 章。
- **细纲准备**：开始本批写作前，建议至少已完成**本批 + 下一批**（共 6 章）的细纲，以避免写下一批时缺细纲；细纲按《5-细纲扩充规范》可分批扩充。

**本批剧情摘要须包含：**
- 本批章节范围（如第 X–Y 章）；
- 主要情节推进（2–5 条）；
- 关键人物状态/关系变化（男女主及重要配角）；
- 与后续批次相关的伏笔或未解悬念；
- **本批爽点/钩子简要**（每章爽点类型、章尾钩子类型）；
- **时间线/场景线**（本批涉及的时间跨度、主要场景顺序）。

**自检与提示：**  
创作时需确认：是否按分批执行、每批是否为 3 章、每批结束后是否生成了剧情摘要。

## 二、对话写作规范

### 2.1 对话引用格式（硬性要求）

**正文中所有对话必须使用双引号 ""，不得使用直角引号「」。** 以符合番茄小说等平台的对话格式规范。

- 正确示例："那个……要演多久啊？总不能一直演下去吧。"
- 错误示例：「那个……要演多久啊？总不能一直演下去吧。」

创作时自始至终统一使用双引号 ""，便于正文直接复制粘贴至番茄小说发布。

### 2.2 对话设计原则

```markdown
✅ 番茄式对话特征：
1. 短！短！短！（一句话不超过15字）
2. 直接！不绕弯子！
3. 有火药味/暧昧味/张力
4. 每句都是武器/糖弹
5. 必须有动作/表情/心理配合
6. 引用格式：一律使用双引号 ""
```

### 2.3 对话公式模板

#### 公式1：冲突对话（对话内容用双引号 ""）
```
人物A：[挑衅/指责/命令]
  - 语气：尖锐/冷漠/强势
  - 动作：[配合动作]
  - 表情：[配合表情]

人物B：[反击/嘲讽/回击]
  - 语气：冷静/尖锐/讽刺
  - 动作：[配合动作]
  - 表情：[配合表情]
  - 心理：[内心OS]

人物A：[被噎住/更生气]
  - 语气：[变化]
  - 动作：[变化]

人物B：[致命一击]
  - 语气：[冷漠/嘲弄]
  - 内容：[让人物A说不出话的话]
```

#### 公式2：暧昧对话（对话内容用双引号 ""）
```
男主：[试探/撩拨]
  - 语气：[低沉/温柔/危险]
  - 动作：[靠近/触碰/注视]
  - 距离：[拉近距离]

女主：[慌乱/害羞/装傻]
  - 语气：[结巴/强装镇定]
  - 生理反应：[脸红/心跳加速/手心出汗]
  - 心理：[内心OS：救命/啊啊啊]

男主：[更进一步]
  - 动作：[更亲密的动作]
  - 语言：[更撩人的话]

女主：[彻底慌乱/逃跑]
```

#### 公式3：威胁对话（对话内容用双引号 ""）
```
反派：[威胁/警告]
  - 语气：[阴冷/威胁]
  - 动作：[逼近/施压]

女主/男主：[淡定/反击]
  - 语气：[毫无畏惧]
  - 动作：[不退反进]
  - 回击：[更狠的话]

反派：[被震慑]
```

### 2.4 对话节奏控制

```markdown
快节奏对话（冲突场景）：
> "你签不签？"
> "不签。"
> "你必须签！"
> "我不签你能怎样？"
> [动作：摔东西/拉住她]
> "放手！"
> "我不放！"

慢节奏对话（暧昧场景）：
> "你……"她咬了咬嘴唇。
> "嗯？"他低低应声，声音在耳边。
> "能不能……离我近点？"
> "这样近吗？"
> 他突然靠近，鼻尖几乎碰到她的。
> "太……太近了……"
> "那这样呢？"
```

### 2.5 对话标签替代词

```markdown
不要总是用"说"，可以替换为：

**微笑类：**
- "……"他嘴角勾起
- "……"他眼弯了弯
- "……"他忍俊不禁

**思考类：**
- "……"他沉思片刻
- "……"他若有所思

**疑问类：**
- "……"他挑眉
- "……"他反问
- "……"他不解

**认真类：**
- "……"他看着她，认真道
- "……"语气肯定
- "……"声音低沉

**笃定类：**
- "……"他笃定道
- "……"毫无疑问

**无奈类：**
- "……"他叹了口气
- "……"他扶额
- "……"拿她没办法

**或者直接用动作：**
- "……"他拍了拍她的头
- "……"她转身就走
- "……"他递过来一杯水
```

## 三、爽点设计规范

### 3.1 爽点类型

#### 类型1：打脸爽
```
定义：女主被轻视/嘲讽，然后反击，打脸众人

设计要点：
1. 铺垫：反派先嚣张
2. 转折：女主淡定/笑
3. 爆发：展现实力/身份
4. 打脸：众人震惊/反派惨败
5. 爽感：读者爽到飞起

公式：
反派嘲讽 → 女主淡定 → 暗示实力 → 众人不信 → 爆发展示 → 众人震惊 → 打脸成功
```

#### 类型2：身份曝光爽
```
定义：女主隐藏大佬身份曝光

设计要点：
1. 暗示：前面不断暗示
2. 期待：读者期待曝光
3. 爆发：关键时刻身份曝光
4. 震惊：众人震惊
5. 爽感：读者期待得到满足

公式：
身份暗示 → 持续铺垫 → 关键时刻 → 身份曝光 → 众人震惊 → 反派惨败 → 爽感爆发
```

#### 类型3：男主追妻爽
```
定义：男主前期渣/冷漠，后期卑微追妻

设计要点：
1. 前期：男主冷漠/误解女主
2. 转折：发现真相/女主身份
3. 后悔：开始后悔/愧疚
4. 追妻：卑微追妻/被拒
5. 爽感：读者看男主受罪

公式：
前期冷漠 → 发现真相 → 开始后悔 → 卑微追妻 → 屡次被拒 → 越挫越勇 → 爽感满满
```

#### 类型4：宠溺爽
```
定义：男主极致宠女主

设计要点：
1. 女主遇到困难
2. 男主出现
3. 一句话/一个动作解决
4. 宠溺话语/行动
5. 爽感：被宠的感觉

公式：
女主困难 → 男主登场 → 霸气解决 → 宠溺话语 → 众人羡慕 → 爽感
```

#### 类型5：复仇爽
```
定义：报复渣男/绿茶

设计要点：
1. 前期：被欺负/陷害
2. 蓄力：暗中布局
3. 爆发：开始复仇
4. 成功：反派受到惩罚
5. 爽感：大仇得报

公式：
被欺负 → 暗中布局 → 开始复仇 → 环环相扣 → 反派崩溃 → 大仇得报 → 爽感
```

### 3.2 爽点强度分级

```markdown
小爽点（3-5章一个）：
- 强度：⭐⭐⭐
- 时长：1-2章内完成
- 类型：打脸小配角/小反转/男主护短
- 示例：绿茶嘲讽，女主一句话噎住她
- 爽感：轻爽

中爽点（10-15章一个）：
- 强度：⭐⭐⭐⭐
- 时长：3-5章铺垫+爆发
- 类型：身份部分曝光/中型打脸事件/渣男受惩罚
- 示例：宴会上女主身份部分曝光，众人震惊
- 爽感：中爽

大爽点（30章一个）：
- 强度：⭐⭐⭐⭐⭐
- 时长：10章以上铺垫+爆发
- 类型：身份完全曝光/最大型打脸/终极复仇
- 示例：女主真实身份完全曝光，全城震惊，渣男跪求复合被拒
- 爽感：大爽
```

### 3.3 爽点设计公式

```
通用公式：
压抑（被欺负/被轻视/被误解）
  ↓
暗示（女主不简单/有后手）
  ↓
铺垫（持续铺垫，增加期待）
  ↓
爆发（关键时刻，爽点爆发）
  ↓
震惊（众人震惊/反派崩溃）
  ↓
爽感（读者爽感爆发）
```

## 四、钩子设计规范

### 4.1 钩子类型

#### 类型1：悬念式钩子
```
定义：提出疑问，让读者想知道答案

设计要点：
- 问题要有趣
- 不能立即揭晓
- 要有期待感

示例：
"林浅接起电话，那头传来熟悉的声音——
'浅浅，是你吗？'
林浅浑身僵硬！
这个声音……
怎么和死去的那个男人，一模一样？！"
```

#### 类型2：反转式钩子
```
定义：意料之外的发展

设计要点：
- 前面铺垫A
- 结果是B
- 反转要有冲击力

示例：
"她以为那个孩子是顾寒川的。
直到亲子鉴定报告出来——
不是。
那……是谁的？"
```

#### 类型3：危机式钩子
```
定义：突发危险/困境

设计要点：
- 危机要突然
- 要让人担心
- 要想知道如何解决

示例：
"就在这时——
一把刀刺了过来！
林浅想躲，但来不及了！
千钧一发之际——
一只手挡住了！
血……
满地都是血！"
```

#### 类型4：身份式钩子
```
定义：暗示/揭露身份

设计要点：
- 身份要震撼
- 要有铺垫
- 要有期待

示例：
"老陈回头，恭敬道：
'大小姐，陆氏集团已经准备好了。'
林浅笑了。
（顾寒川，你以为赶走的是一只蚂蚁？
呵呵……你赶走的，是狼。）"
```

#### 类型5：情感式钩子
```
定义：暧昧/吃醋/心动

设计要点：
- 情感要强烈
- 要有张力
- 要让人嗑CP

示例：
"他的呼吸在她耳边。
'林浅，'声音低哑，'你逃不掉的。'
心跳……
漏了一拍。
又跳了一拍。
然后——
彻底乱了节奏。"
```

### 4.2 钩子设计公式

```
公式1：层层递进式
第一层：小悬念
第二层：暗示
第三层：即将揭晓
第四层：意外反转

公式2：戛然而止式
事件发生 → 就在关键时刻 → 戛然而止 → 留下悬念

公式3：信息差式
读者知道，角色不知道 → 期待角色发现 → 发现时的反应
```

### 4.3 钩子位置规则

```markdown
✅ 钩子位置：
1. 章节开头（可选）：承接上章，制造小悬念
2. 章节中间（高频）：小钩子，保持兴趣
3. 章节结尾（必须）：大钩子，让人想看下一章

❌ 不要：
- 不要每段都有钩子（会疲劳）
- 不要钩子太弱（没效果）
- 不要钩子不解（会烂尾）
```

## 五、AI执行检查清单

**创作章节时，AI必须自检：**

### 对话检查
- [ ] 所有对话使用双引号 ""，未使用直角引号「」
- [ ] 对话短而有力
- [ ] 对话有动作/表情配合
- [ ] 对话推动情节
- [ ] 对话有张力
- [ ] 不过度使用"说"字

### 爽点检查
- [ ] 如果是爽点章节，必须有爽点
- [ ] 爽点明确
- [ ] 爽感强烈
- [ ] 读者能感到爽
- [ ] 爽点强度符合定位

### 钩子检查
- [ ] 章尾必须有钩子
- [ ] 钩子类型明确
- [ ] 钩子够强
- [ ] 让人想看下一章
- [ ] 钩子连贯不矛盾

### 整体检查
- [ ] 正文字数不少于4000字（不含标点）
- [ ] 正文为纯叙述：未出现 "### 场景X：xxx"、"#### 开场/发展/高潮/结尾"；未使用「---」分隔线；对话均为双引号 ""
- [ ] 对话占比超过40%
- [ ] 节奏快不拖沓
- [ ] 符合番茄爆款风格
- [ ] 按照输出模板输出

### 分批生成检查
- [ ] 本批仅创作 3 章，未一次性写更多章
- [ ] 本批全部完成后已输出「本批剧情摘要」
- [ ] 下一批创作时已把「上一批剧情摘要」作为上下文使用

## 六、使用提示

**给用户的提示：**
1. AI 按「每批 3 章」创作，不会一次性写完所有章节
2. 每批结束后会输出「本批剧情摘要」，请将摘要提供给下一批对话作为上下文
3. AI 会基于本规范 + 当前批次细纲 + 上一批剧情摘要（若有）创作
4. 会按照《小说章节输出模板.md》输出；若不满意可要求修改

**给AI的提示：**
1. 必须严格按照本规范创作，且按「分批生成」执行：每批仅 3 章
2. **字数**：每章正文（不含标点）**必须 ≥4000 字**。写完后先自行统计正文字数（去除标点）；若不足 4000 字，**必须在本章内补写至达标后再输出**，不得以「篇幅所限」等理由输出不足 4000 字的章节
3. **人名与场景**：正文中的人名、场景/地点名须与细纲及人物设定一致，**不得使用对标短篇小说中出现的任何人名或场景名**
4. 必须应用所有专项规范（对话、爽点、钩子）
5. 必须按《小说章节输出模板.md》输出正文须为**可直接复制粘贴使用的纯叙述**：不得出现 "### 场景X：xxx"、"#### 开场/发展/高潮/结尾" 等场景标题或小标题；**对话一律使用双引号 ""**，不得使用直角引号「」（符合番茄小说平台）；场景与段落之间仅用空行分隔，不得使用「---」分隔线
6. 本批 3 章全部写完后，必须输出「本批剧情摘要」
7. 创作后必须自检（含**每章正文字数是否≥4000字不含标点**、分批、剧情摘要、正文格式）
8. 输出格式必须统一
