# AI选题生成规范文档

> 版本：v1.0
> 创建日期：2026-02-15
> 用途：基于短篇小说生成番茄爆款现言选题
> 目标字数：约60-80万字（每章不少于4000字不含标点，150-200章）

---

## 一、使用说明

### 1.1 文档用途
当用户提供一篇短篇小说后，AI将基于本规范文档，为用户创作一个新的**番茄爆款现言选题**及详细设定。

### 1.2 使用流程
```
用户输入：短篇小说文本
  ↓
AI执行：基于本规范生成选题
  ↓
AI输出：按《2-选题输出模板》输出选题卡片（选题卡片为后续人设、大纲、细纲、写章等步骤的唯一输入）
```

### 1.3 核心原则
```markdown
1. 必须符合番茄爆款现言风格
2. 必须包含强烈冲突和爽点
3. 必须有明确的黄金三章设计
4. 必须有鲜明的人设
5. 目标字数：约60-80万字
6. 【命名与场景】长篇中所有人物姓名、场景/地点名称**不得与对标短篇小说中的人名、场景重复**；可复用短篇的人物关系模式、情节套路、情感细节，但**人名与具体场景名须全部新创或替换**，不得照搬短篇中的任何角色名或地点名。
```

## 二、AI执行指令

### 2.1 输入分析

**第一步：分析原短篇小说**
```
当用户提交短篇小说后，AI需要：

□ 识别原小说的核心情感线
  - 主线是什么？（爱情/复仇/成长）
  - 情感基调？（甜宠/虐恋/轻松）

□ 提取可复用元素（仅限结构/套路，不沿用具体名称）
  - 人物关系模式（可复用，但须为新人物命名）
  - 冲突类型
  - 情节套路
  - 亮点和特色

□ 明确不可复用的内容（必须新创）
  - 短篇中出现的**所有人名**：不得在长篇中直接使用，须另起新名
  - 短篇中出现的**所有场景/地点名**（如学校名、小区名、店名、具体建筑名等）：不得在长篇中直接使用，须另起新名或泛化表述

□ 识别不适合番茄的元素
  - 过于文青的描写
  - 过于慢热的节奏
  - 过于含蓄的情感
  - 需要改进的地方
```

### 2.2 选题生成规则

**AI必须按照以下规则生成新选题：**

#### 核心设定（必须生成）

```markdown
1. 书名（3-5个备选）
   要求：
   - 直白、有爽点
   - 包含核心冲突
   - 有吸引力
   示例：《离婚后，前夫跪求复合》《带球跑：五年后她携萌宝归来炸翻全城》

2. 核心标签（选择2-3个）
   必选标签：
   □ 萌宝 □ 豪门 □ 带球跑 □ 逆袭 □ 打脸
   □ 重生 □ 复仇 □ 甜宠 □ 追妻火葬场

3. 一句话梗概
   要求：
   - 一句话说清故事
   - 包含核心冲突
   - 有爽点暗示
   示例："五年前她被渣男抛弃，带着孩子远走他乡；五年后她华丽归来，身份曝光惊呆所有人！"

4. 核心卖点（3-5个）
   必须包含：
   - 爽点类型（打脸/逆袭/宠溺）
   - 情感钩子（追妻火葬场/破镜重圆）
   - 身份反差（隐藏大佬身份）
   - 萌宝助攻（如果有）
```

#### 人设生成（必须详细）

**女主角设定表（AI必须填写）：**
```
【基本信息】
- 姓名：必须好记、有美感；**不得与对标短篇小说中任何人物同名**
- 年龄：23-28岁
- 身高：165-170cm（番茄女主标准身高）
- 外貌特征：
  * 整体：美到有记忆点（如：肤白如雪、红唇似火）
  * 特殊标记：胎记/疤痕/异瞳（增加记忆点）
  * 穿搭风格：前期朴素/后期华丽

【身份设定】（必须有反差）
- 表面身份：
  * 选项：落魄千金/单亲妈妈/普通人/被打压的原配
  * 状态：被轻视/被欺负/处于低谷
- 真实身份：
  * 必须是大佬！（陆氏千金/神秘投资人/黑客Z/神医传人）
  * 至少2个隐藏身份
  * 身份曝光时间规划（第X章部分曝光/第Y章完全曝光）

【性格设定】
- 核心性格关键词（3-5个）：
  * 必须：腹黑/聪明/护短/有底线
  * 可选：佛系/戏精/作精/冷艳
- 性格反差：
  * 表面：柔弱/佛系/无所谓
  * 真实：满级大佬/睚眦必报
- 口头禅（1-2句）：
  * 要有记忆点
  * 体现性格

【金手指/能力】
- 从原小说提取/新设计：
  * 商业才能
  * 黑客技术
  * 医术
  * 设计天赋
  * 其他特殊技能

【情感经历】
- 前情：和男主的关系（前任/夫妻/陌生人）
- 状态：离婚/分手/被迫分离/契约婚姻
- 心理：还爱/不爱/爱恨交织
```

**男主角设定表（AI必须填写）：**
```
【基本信息】
- 姓名：必须有质感、好记；**不得与对标短篇小说中任何人物同名**
- 年龄：27-32岁（比女主大3-5岁）
- 身高：185cm以上
- 外貌特征：
  * 整体：A爆了、气场强大
  * 五官：立体深邃、有压迫感
  * 穿搭：西装革履/贵气逼人

【身份设定】
- 核心身份：
  * 必须是顶级大佬！
  * 顾氏/陆氏/傅氏等集团总裁
  * 掌控半个商界/整个行业

【性格设定】
- 核心性格关键词（3-5个）：
  * 必须：高冷/禁欲/强势/护短
  * 后期变化：追妻火葬场
- 对女主的特殊：
  * 唯一例外
  * 双标（对别人冷漠，对女主……）
- 口头禅（1-2句）：
  * 要苏、要撩

【情感态度】
- 前期：冷漠/误会/利用/契约
- 转折点：第X章开始动心/后悔
- 后期：追妻/宠妻/偏执
```

**男二/女配设定（可选，但推荐）：**
```
【男二（深情守护型）】
- 姓名、身份、性格
- 对女主的感情：默默守护/默默付出
- 结局：祝福/离开/找到真爱

【反派女配（绿茶型）】
- 姓名、身份
- 与女主的关系：情敌/姐妹/闺蜜
- 性格：表面无辜/内心恶毒
- 下场：身败名裂/得到惩罚

【助攻团】
- 闺蜜/兄弟/萌宝/管家等
- 作用：助攻/搞笑/提供信息
```

**生成选题后，AI必须自检：**

### 基础检查
- [ ] 生成了3-5个备选书名
- [ ] 选定了2-3个核心标签
- [ ] 写了一句话梗概
- [ ] 规划了3-5个核心卖点
- [ ] 详细设定了男女主角

### 人设检查
- [ ] 女主有身份反差（表面vs真实）
- [ ] 女主至少2个隐藏大佬身份
- [ ] 男主是顶级大佬
- [ ] 人设鲜明有记忆点
- [ ] 有口头禅/特殊标记

### 情节检查
- [ ] 规划了完整的三幕式结构（150-200章）
- [ ] 第1章有强冲突
- [ ] 第3章有爽点爆发
- [ ] 爽点密度合格（3-5章一小爽）
- [ ] 有身份曝光规划

### 黄金三章检查
- [ ] 第1章前500字有强冲突
- [ ] 第2章男主正式登场
- [ ] 第3章爽点爆发
- [ ] 每章结尾有钩子

### 番茄风格检查
- [ ] 符合番茄爆款现言风格
- [ ] 有强烈冲突
- [ ] 有密集爽点
- [ ] 有鲜明人设
- [ ] 目标字数约60-80万字

## 五、使用提示

**给用户的提示：**
1. 本规范规定如何生成选题；交付物须按《2-选题输出模板》输出为**选题卡片**，后续所有步骤（人设、大纲、细纲、写章）均以选题卡片为输入。
2. 提交短篇小说时，可以附加要求（如：想要萌宝/想要追妻火葬场）
3. 如果对生成的选题不满意，可以要求AI调整
4. 选题卡片确认后，进入下一步：人物设定

**给AI的提示：**
1. 必须严格按照本规范生成选题
2. 不能遗漏任何必须项
3. 选题必须符合番茄爆款风格
4. 必须详细，不能太简略
5. 生成后必须自检
