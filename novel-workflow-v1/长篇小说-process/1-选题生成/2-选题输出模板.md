# 选题卡片输出模板规范

> 版本：v1.0
> 创建日期：2026-02-15
> 用途：AI生成选题后，按照本模板输出标准化的选题卡片
> 目标：输出格式统一、信息完整的选题文档

---

## 一、使用说明

### 1.1 文档用途
当AI完成选题分析后，必须按照本模板输出选题卡片，确保输出格式统一、信息完整。

### 1.2 使用时机
```
AI按《1-选题生成规范》分析短篇小说并生成选题 → 按照本模板输出选题卡片
```
**说明：** 选题卡片是选题的标准化输出，后续人物设定、大纲、细纲、章节创作等步骤均以**选题卡片**为输入。

## 附录：番茄爆款文案结构

### 结构01：强冲突开场型
**结构：** 强冲突开头 + 女主反击 + 身份暗示 + 悬念钩子

**原理：**
第一章必须有强冲突（离婚/背叛/抛弃），女主不能软弱，必须有反击，最后留下身份暗示的钩子。这是番茄最有效的开场方式。

**案例模板：**
1. **强冲突开头**: 离婚协议甩在脸上，渣男冷漠决绝
   - **女主反击**: 签得飞快，潇洒离开，暗示有阴谋
   - **身份暗示**: 接到神秘电话/神秘人物出现
   - **悬念钩子**: 她到底什么身份？渣男会后悔吗？

---

### 结构02：带球跑归来型
**结构：** 时间跳跃 + 华丽归来 + 萌宝助攻 + 首次打脸

**原理：**
五年时间跨度，女主携萌宝华丽归来，形象大变，首次打脸成功，萌宝展现实力或助攻，读者爽感强烈。

**案例模板：**
1. **时间跳跃**: 五年后，机场/宴会厅
   - **华丽归来**: 形象大变，气场全开，众人惊艳
   - **萌宝助攻**: 萌宝出现，戏精/腹黑/天才
   - **首次打脸**: 绿茶嘲讽，女主轻松打脸

---

### 结构03：重生复仇型
**结构：** 重生醒悟 + 时间回溯 + 改写命运 + 首次打脸

**原理：**
女主前世惨死，重生回到关键节点，开始改写命运，报复渣男，首次打脸成功，读者期待后续。

**案例模板：**
1. **重生醒悟**: 前世惨死，重生回XX年前
   - **时间回溯**: 回到关键时间点（结婚前/背叛前）
   - **改写命运**: 不再是原来的傻白甜，开始反击
   - **首次打脸**: 改变前世某个被欺负的场景

---

### 结构04：契约婚姻型
**结构：** 被迫结婚 + 男主误会 + 契约生活 + 日久生情

**原理：**
女主被迫嫁给"残疾/可怕"男主，发现男主不残疾/很帅，契约恋爱，假戏真做，日久生情，甜蜜温馨。

**案例模板：**
1. **被迫结婚**: 家族危机/被迫替嫁
   - **男主误会**: 以为她是拜金女/心机女
   - **契约生活**: 同居生活，斗智斗勇
   - **日久生情**: 男主发现女主的好，开始动心

---

### 结构05：认错人型
**结构：** 一夜情 + 认错人 + 怀孕/带球跑 + 再次相遇

**原理：**
女主酒后/意外与陌生男人一夜情，认错人/怀孕消失，多年后携萌宝归来，再次相遇，男主认出/亲子鉴定。

**案例模板：**
1. **一夜情**: 酒店房间/认错房间
   - **认错人**: 以为是未婚夫/男友
   - **怀孕/带球跑**: 发现怀孕，远走他乡
   - **再次相遇**: 多年后，男主认出/萌宝长得像男主

## 标题与简介优化规范

### 一、标题设计规则

#### 1.1 番茄爆款标题特征

```markdown
✅ 好的标题特征：
- 直白：一眼看出是讲什么的
- 有爽点：包含核心爽点
- 有冲突：包含核心冲突
- 有钩子：让人想点进去看
- 长度：10-20字最佳

❌ 不好的标题：
- 文艺：太含蓄，看不出爽点
- 晦涩：不知道讲什么
- 太长：超过25字
- 没冲突：平淡无奇
```

#### 1.2 标题公式模板

**公式1：身份反差型**
```
[表面身份] + [真实身份] + [爽点]
示例：
- 《被赶出家门后，我成了首富千金》
- 《离婚后，前夫跪求复合》
- 《带球跑：五年后她携萌宝归来炸翻全城》
```

**公式2：时间跳跃型**
```
[时间跨度] + [归来] + [爽点]
示例：
- 《五年后，她携三胞胎华丽归来》
- 《十年后，落魄千金逆袭成首富》
```

**公式3：复仇逆袭型**
```
[被欺负] + [反击/逆袭] + [爽点]
示例：
- 《被渣后，我转身嫁给了渣叔》
- 《退婚后，我被千亿总裁宠上天》
```

**公式4：萌宝助攻型**
```
[萌宝] + [助攻/认亲] + [爽点]
示例：
- 《天才萌宝：妈咪，那个富豪像我爹地》
- 《一胎三宝：妈咪哪里逃》
```

#### 1.3 标题优化步骤

```markdown
步骤1：列出3-5个备选标题
步骤2：对照特征检查，删除不合格的
步骤3：测试哪个最吸引人
  - 问自己：如果我是读者，我会点哪个？
  - 问朋友：哪个标题最吸引你？
步骤4：确定最终标题
步骤5：如果发布后数据不好，可以修改标题
```

### 二、简介写作规则

#### 2.1 番茄爆款简介特征

```markdown
✅ 好的简介特征：
- 开头炸裂：第一句话就有冲突/爽点
- 情节清晰：让人知道故事大概
- 人设鲜明：主角是什么人
- 爽点明确：有哪些爽点
- 钩子强力：让人想追更
- 长度：200-500字最佳

❌ 不好的简介：
- 铺垫太多：前100字还没进入正题
- 情节模糊：不知道讲什么
- 人设平淡：主角没特色
- 没爽点：平淡无奇
- 没钩子：不想追更
```

#### 2.2 简介公式模板

**模板1：三段式简介**
```
第一段（核心冲突）：100字左右
  - 交代核心冲突（离婚/背叛/抛弃）
  - 女主怎么反击的
  - 留下钩子

第二段（时间跳跃/身份反转）：100字左右
  - 时间跨度（五年后）
  - 女主华丽归来
  - 形象/地位大变

第三段（爽点预告+钩子）：100字左右
  - 核心爽点（身份曝光/打脸/复仇）
  - 男主追妻（如果有）
  - 强力钩子

示例：
第一段：离婚那天，林浅签得飞快。"顾寒川，你会后悔的！"顾寒川冷笑："我这辈子最后悔的就是娶了你！"——殊不知，他后悔的还在后面。

第二段：五年后，机场。一个女人牵着萌宝华丽归来。肤白貌美，气场全开，众人惊艳。绿茶女配嘲讽？呵，她随手掏出陆氏千金黑金卡，全场死寂！

第三段：当她的真实身份曝光——陆氏集团继承人、神秘投资人Q、黑客Z……顾寒川跪在她面前："浅浅，再给我一次机会！"林浅轻笑："顾总，我们不熟。"
```

**模板2：对话式简介**
```
开头用对话直接进入冲突：
"签了它，这一千万就是你的。"
离婚协议甩在林浅脸上。
她签得飞快："顾总，你会后悔的！"
——
五年后，她携萌宝华丽归来。
陆氏千金身份曝光，全城震惊！
前夫跪求复合："浅浅，再给我一次机会！"
她轻笑："抱歉，顾总，我们不熟。"

【萌宝助攻+身份反转+追妻火葬场】
【爽文，无虐，男女主双洁】
```

**模板3：人物标签式简介**
```
【女主】林浅
表面：被抛弃的可怜原配
真实：陆氏千金、神秘投资人Q、黑客Z

【男主】顾寒川
前期：高冷渣男总裁
后期：卑微追妻狗

【核心爽点】
✅ 离婚开场，女主潇洒离开
✅ 五年后华丽归来，身份曝光打脸众人
✅ 追妻火葬场，男主卑微求复合被拒
✅ 萌宝助攻，天才黑客儿子
✅ 打脸爽文，无虐，爽到飞起

【一句话简介】
五年前她被抛弃，携子远走；五年后华丽归来，身份曝光惊呆全城！
```

#### 2.3 简介优化检查清单

```markdown
□ 开头第一句话就有冲突/爽点
□ 情节清晰，让人知道故事大概
□ 人设鲜明，主角有特色
□ 爽点明确，至少3个
□ 有强力钩子，让人想追更
□ 长度在200-500字
□ 没有废话，每句话都有用
□ 符合番茄风格（直白、爽快）
□ 没有敏感内容
□ 标签明确（2-3个）
```

```markdown
  - 文字：书名（醒目）
□ 色调：
  - 甜文：粉/白/暖色调
  - 爽文：黑/金/冷色调
□ 避免：
  - 太花哨
  - 太low
  - 不相关

注：番茄平台可以不要封面，系统会自动生成
```
