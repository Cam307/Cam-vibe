# AI大纲生成规范文档

> 版本：v1.0
> 创建日期：2026-02-15
> 用途：基于选题卡片+人物设定+短篇小说生成完整大纲
> 目标字数：约60-80万字（150-200章，每章不少于4000字不含标点）

---

## 一、使用说明

### 1.1 文档用途
当用户确定选题卡片并完成人物设定后，AI将基于**选题卡片 + 人物设定 + 原始短篇小说**生成一份**详细的完整大纲**（150-200章）。其中选题卡片由《2-选题输出模板》产出，人物设定由《3-人物设定规范》产出，均须在大纲生成之前完成。

### 1.2 使用流程
```
用户输入：选题卡片 + 人物设定 + 原始短篇小说
  （选题卡片见《2-选题输出模板》；人物设定见《3-人物设定规范》）
  ↓
AI执行：基于本规范生成大纲
  ↓
AI输出：完整的章节大纲（150-200章）
```

### 1.3 核心原则
```markdown
1. 必须包含所有章节的简要大纲
2. 必须规划所有爽点和钩子
3. 必须明确每章的情节推进
4. 必须标注所有重要节点（身份曝光、表白、打脸等）
5. 必须符合三幕式结构
```

## 二、AI执行指令

### 2.1 输入分析

**第零步：分析人物设定**
```
AI需要提取（来自《3-人物设定规范》产出的人物设定文档）：
□ 主角与重要配角的姓名、身份、关系
□ 人物成长阶段与关键节点
□ 人物关系网络与结局设定

并确保：
□ 大纲中使用的称呼、人物列表与人设文档一致
□ 重要节点（身份曝光、感情转折等）与人设中的成长线、曝光时间一致
```

**第一步：分析选题卡片**
```
AI需要提取：
□ 核心标签和卖点
□ 男女主人设
□ 情节主线规划
□ 黄金三章设计
□ 爽点规划

并确认：
□ 目标字数：约60-80万字
□ 目标章节：150-200章
□ 核心冲突是什么
□ 核心爽点是什么
```

**第二步：分析原始短篇小说**
```
AI需要提取：
□ 可复用的情节套路
□ 可复用的对话模式
□ 可复用的情感细节
□ 可复用的场景描写（仅限类型与氛围，不沿用具体场景名/地点名）
□ 需要改造的部分

并决定：
□ 哪些情节可以直接使用（须替换为人设中的新人物名、新场景名）
□ 哪些情节需要改造加速
□ 哪些情节需要删除（不符合番茄）

【硬性约束】大纲中出现的所有人名、场景/地点名称**不得与原始短篇小说中的人名、场景重复**；须使用人物设定与选题卡片中已确定的新名称。
```

### 2.2 大纲生成规则

**AI必须按照以下规则生成大纲：**

#### 总体结构

```markdown
【总体布局】
- 总字数：约60-80万字
- 总章节：150-200章
- 平均每章：不少于4000字（不含标点）（全书总字数约60-80万字若150-200章）

【三幕式结构分配】
- 第一幕：约50章（10万字）- 矛盾激化
- 第二幕：约75章（15万字）- 高潮迭起
- 第三幕：约50章（10万字）- 圆满结局

【章节类型】
- 情节推进章：70%（推动主线）
- 感情升温章：20%（男女主互动）
- 配角/过渡章：10%（铺垫/转折）
```

#### 每章大纲格式

**AI必须为每一章生成以下内容：**

```
第X章：[章节标题]

【本章定位】

- 承上启下：[承接上章什么 + 开启下章什么]

【核心情节】
- 开头（200-300字）：[如何承接上章，制造悬念]
- 发展（1500字）：[主要情节，2-3个场景]
- 高潮（500字）：[情绪爆发/冲突激化/爽点实现]
- 结尾（200-300字）：[本章小结 + 章尾钩子]

【场景拆分】
场景1：[地点] + [人物] + [情节]
场景2：[地点] + [人物] + [情节]
场景3：[地点] + [人物] + [情节]

【爽点/钩子】
- 本章爽点：[如果有，描述爽点类型]
- 章尾钩子：[必须！描述钩子类型]

【人物状态】
- 女主：[情绪/心理/处境变化]
- 男主：[情绪/心理/态度变化]
- 重要配角：[如有必要]

【字数预计】
- 预计字数：不少于4000字（不含标点）
```
