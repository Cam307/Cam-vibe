# AI细纲扩充规范文档

> 版本：v1.1
> 创建日期：2026-02-15
> 用途：基于大纲+选题，为每一章生成详细细纲
> 目标：让AI能够直接根据细纲写出完整章节

---

## 一、使用说明

### 1.1 文档用途
当用户确定大纲后，AI将基于完整大纲和选题卡片，为**每一章生成详细的细纲**。

### 1.2 使用流程
```
用户输入：完整大纲 + 选题卡片 + 章节号（或本批章节号范围）
  ↓
AI执行：基于本规范生成该章/本批的详细细纲
  ↓
AI输出：该章/本批的详细细纲（可直接用于写作）
```

**与分批写作的配合：** 细纲可分批扩充（例如先扩充第 1–3 章，再扩充第 4–6 章）。若章节内容将按「每批 3 章」分批生成，建议至少先完成当前批次及下一批次所需的细纲，以保证情节与人设衔接顺畅。

### 1.3 核心原则
```markdown
1. 必须极其详细，不能遗漏任何细节
2. 必须包含所有对话的要点
3. 必须包含所有场景的描写要点
4. 必须标注所有情绪转折点
5. 必须符合番茄爆款风格
6. 【命名与场景】细纲中使用的**人名、场景/地点名**须与大纲及人物设定一致，**不得与对标短篇小说中的人名、场景重复**；若大纲与人设已按要求新创名称，细纲严格沿用即可。
```

## 二、AI执行指令

### 2.1 输入分析

**第一步：分析章节在大纲中的位置**
```
AI需要确认：
□ 这是第几章
□ 在三幕式结构中的位置（开篇/发展/高潮/结局）
□ 章节类型（情节推进/感情升温/配角过渡）
□ 承接上章什么
□ 开启下章什么
□ 本章是否有重要节点
□ 本章是否有爽点
□ 本章钩子是什么
```

**第二步：分析本章涉及的人物**
```
AI需要确认：
□ 本章出场人物（所有）
□ 每个人物的情绪状态
□ 每个人物的目的/动机
□ 人物之间的关系变化
```

**第三步：分析本章的情节结构**
```
AI需要确认：
□ 开头（200-300字）：如何承接上章
□ 发展（1500字）：几个场景，每个场景的情节
□ 高潮（500字）：情绪爆发/冲突激化
□ 结尾（200-300字）：本章小结 + 章尾钩子
```
