# AI全面检查规范文档

> 版本：v3.0
> 创建日期：2026-02-15
> 用途：按周期对已写章节做全面检查，兼顾质量与执行成本
> 核心变化：v3.0 改为「每写满 5 章做一次全面检查」+ 关键节点必检；支持增量检查

---

## 一、使用说明

### 1.1 文档用途
在章节创作过程中，按**检查周期**对已写内容进行**全面检查**，确保人设、情节、钩子等前后一致，避免问题累积。每章写完后仍须按《8-自我检查规范》做本章自检。

### 1.2 检查周期与方式

```markdown
【全面检查触发时机】
- 每写满 5 章：做一次第 1~N 章全面检查（如写完第 5、10、15、20…章后）
- 关键节点必检：写完第 3 章后必须做一次（黄金三章专项 + 全量）；建议在第 50、100 章等节点再做全量检查

【非全检轮次（可选）】
- 若在「未满 5 章」且非关键节点时仍需检查，可采用**增量检查**：只详细检查最近 5 章 + 与前文的人设/情节/钩子一致性，不做 1~N 章逐章全量统计
```

### 1.3 使用流程
```
AI写完第N章
  ↓
先按《8-自我检查规范》做本章自检（每章必做）
  ↓
判断：是否到达全面检查节点？（N=3 或 N 为 5 的倍数，或用户指定节点）
  ↓
若到达：基于本规范全面检查第 1~N 章，输出全面检查报告
若未到达：可跳过，或做增量检查（最近 5 章 + 一致性）
  ↓
用户确认/修改 → 继续写第 N+1 章
```

### 1.4 与自检的关系
- **每章自检**（《8-自我检查规范》）：每写完一章必做，只检查本章。
- **全面检查**（本规范）：按周期或关键节点做，检查 1~N 章整体。两者配合使用，先自检再视情况做全检。

## 二、AI执行指令

### 2.1 检查范围

**全量全面检查时（每 5 章或关键节点）：**
```
□ 第1章到第N章的所有章节
□ 本章（第N章）的详细检查
□ 前面所有章节的一致性检查
□ 人设一致性（第1-N章）
□ 情节连贯性（第1-N章）
□ 钩子连贯性（第1-N章）；若有《4-大纲生成规范》产出的钩子清单或钩子分布总表，可依据其检查未回收钩子及回收是否到位
□ 如果N≤3，执行黄金三章专项检查
```

**增量检查时（可选，未到全检节点）：**
```
□ 最近 5 章的详细检查
□ 最近 5 章与更前章节的人设/情节/钩子一致性
□ 不要求对第 1~(N-5) 章逐章重新统计
```

---

### 2.2 检查流程

**第一步：统计性检查（数据化）**
```
□ 第1-N章的总字数统计
□ 平均每章字数
□ 对话占比统计
□ 段落长度统计
□ 爽点数量统计
□ 钩子数量统计
□ 人物出场统计
```

**第二步：质量性检查（深度分析）**
```
□ 节奏评估：是否过快/过慢/拖沓
□ 语言评估：是否符合番茄风格
□ 情绪评估：是否够烈
□ 爽点密度：是否达标
□ 钩子连贯性：是否连贯
□ 人设一致性：是否OOC
□ 逻辑连贯性：是否有漏洞
```

**第三步：对比性检查（与大纲对比）**
```
□ 是否按大纲执行
□ 是否偏离主线
□ 是否遗漏重要情节
□ 是否遗漏重要爽点
□ 是否遗漏重要钩子
```

**第四步：专项检查（根据章节号）**
```
□ 如果N≤3：执行黄金三章专项检查（详见第三章）
□ 如果N>3：执行常规检查
```

**第五步：生成全面检查报告**
```
□ 给出总体评分
□ 列出所有问题
□ 给出优化建议
□ 提供修改方案
□ 给出下一章建议
```

## 三、黄金三章专项检查（重要！）

### 3.1 黄金三章定义

```markdown
黄金三章 = 第1章 + 第2章 + 第3章

重要性：
- 决定生死（完读率）
- 决定推荐（算法推荐）
- 决定追读（读者是否继续）

字数与节奏：在满足「前 500 字强冲突」的前提下，单章正文字数须不少于 4000 字（不含标点）；若因节奏需要微调，可在本章备注中说明理由。
```

### 3.2 黄金三章检查清单

**当N≤3时，AI必须执行以下专项检查：**

#### 第1章专项检查

```markdown
✅ 必须检查项：

1. 开头前500字
   - [ ] 是否有强冲突（离婚/背叛/抛弃）
   - [ ] 是否炸裂（不能慢热）
   - [ ] 是否直接进入正题（不能铺垫）
   - [ ] 读者是否会想继续看

2. 女主表现
   - [ ] 是否软弱（不能软弱！）
   - [ ] 是否有反击（必须有！）
   - [ ] 是否哭哭啼啼（不能！）
   - [ ] 是否让人心疼又佩服

3. 男主表现
   - [ ] 是否出场（必须出场或强烈暗示）
   - [ ] 气场是否够强
   - [ ] 态度是否明确（冷漠/渣）
   - [ ] 是否有让人讨厌的地方（为追妻留余地）

4. 爽点
   - [ ] 是否有小爽点（女主反击）
   - [ ] 爽感是否强烈
   - [ ] 读者是否会感到爽

5. 钩子
   - [ ] 章尾是否有钩子（必须！）
   - [ ] 钩子是否够强
   - [ ] 是否想看第2章

6. 整体
   - [ ] 正文字数不少于4000字（不含标点）
   - [ ] 节奏够快
   - [ ] 语言直白不文青
   - [ ] 符合番茄爆款风格

❌ 常见问题：
- 开头铺垫太多
- 女主软弱哭闹
- 没有爽点
- 章尾没有钩子
- 节奏拖沓
```

#### 第2章专项检查

```markdown
✅ 必须检查项：

1. 男主登场
   - [ ] 男主是否正式登场（必须！）
   - [ ] 气场描写是否够A
   - [ ] 是否让人记住
   - [ ] 是否有压迫感

2. 女主表现
   - [ ] 是否有变化（形象/状态）
   - [ ] 是否展现实力/身份暗示
   - [ ] 是否让人期待

3. 互动
   - [ ] 男女主是否有第一次交锋/互动
   - [ ] 是否有张力
   - [ ] 是否有暧昧（可选）

4. 身份暗示
   - [ ] 是否暗示女主身份不简单
   - [ ] 暗示是否自然
   - [ ] 是否让人好奇

5. 钩子
   - [ ] 章尾是否有钩子（必须！）
   - [ ] 钩子是否够强
   - [ ] 是否想看第3章

6. 整体
   - [ ] 承接第1章自然
   - [ ] 推进了情节
   - [ ] 正文字数不少于4000字（不含标点）
   - [ ] 符合番茄爆款风格

❌ 常见问题：
- 男主气场不够
- 没有身份暗示
- 男女主没有互动
- 章尾没有钩子
```

#### 第3章专项检查

```markdown
✅ 必须检查项：

1. 爽点爆发
   - [ ] 是否有爽点爆发（必须！）
   - [ ] 爽点是否明确（打脸/反转）
   - [ ] 爽感是否强烈
   - [ ] 读者是否会爽到飞起

2. 众人反应
   - [ ] 是否有众人震惊的描写
   - [ ] 震惊是否强烈
   - [ ] 反派是否惨败

   - [ ] 绑定是否合理

   - [ ] 男女主关系是否初步建立
   - [ ] 关系是否有发展
   - [ ] 是否让人期待后续

5. 钩子
   - [ ] 章尾是否有钩子（必须！）
   - [ ] 钩子是否够强
   - [ ] 是否想看第4章

6. 整体
   - [ ] 三章节奏递进
   - [ ] 三章有连贯性
   - [ ] 三章完成绑定任务
   - [ ] 符合番茄爆款风格

❌ 常见问题：
- 没有爽点爆发
- 众人反应不够强烈
- 男女主没有绑定
- 章尾没有钩子
```

## 四、常规章节检查（N>3）

### 4.1 检查项目清单

```markdown
当N>3时，AI执行常规检查：

□ 本章（第N章）检查
  - 字数：正文字数不少于4000字（不含标点）
  - 节奏：快而不拖沓
  - 语言：直白不文青
  - 爽点：如果有爽点章节
  - 钩子：章尾必须有

□ 人设一致性检查（第1-N章）
  - 女主：不OOC，性格一致
  - 男主：不OOC，性格一致或合理转变
  - 配角：人设统一

□ 情节连贯性检查（第1-N章）
  - 时间线：准确连贯
  - 逻辑：无漏洞
  - 前后呼应：有呼应

□ 钩子连贯性检查（第1-N章）
  - 所有钩子：有记录
  - 钩子回收：及时回收
  - 钩子连贯：环环相扣

□ 爽点密度检查（第1-N章）
  - 小爽点：每3-5章一个
  - 中爽点：每10-15章一个
  - 大爽点：每30章一个
  - 密度：达标

□ 进度检查（第1-N章）
  - 是否按大纲执行
  - 是否偏离主线
  - 是否遗漏重要情节
```

## 八、使用提示

**给用户的提示：**
1. 每写完一章，AI会自动进行全面检查
2. 检查范围是第1-N章所有章节
3. 前3章会有黄金三章专项检查
4. 如果有问题，AI会给出修改方案
5. 修改后AI会重新检查

**给AI的提示：**
1. 必须在写完每一章后立即检查
2. 检查范围是第1-N章所有章节
3. 不能遗漏任何检查项
4. 如果N≤3，必须执行黄金三章专项检查
5. 必须准确识别所有问题
6. 必须给出合理的修改方案
