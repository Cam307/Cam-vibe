# 短篇小说改写规范（策略与要求）

> 版本：v1.4
> 创建日期：2026-02-15
> 用途：基于输入的短篇小说，制定改写策略
> 核心要求：改写后不得与原小说高度一致

---

## 一、使用说明

### 1.1 文档用途
当用户提供一篇短篇小说后，AI将基于本规范制定**改写策略**，确保：
1. **内容生动鲜活，符合番茄小说平台爆款写作风格**（有画面感、对话有张力、情绪饱满、节奏抓人、每章有钩子有爽点；忌空泛叙述、平淡流水账）
2. 符合番茄爆款短篇小说风格
3. 与原小说有显著差异（避免平台处罚）
4. 保持核心情感，但彻底改写

### 1.2 使用流程
```
用户输入：短篇小说原文
  ↓
AI执行1：分析原小说
  ↓
AI执行2：制定改写策略（使用临时文档，仅用于本次改写；**必须写明目标正文字数、计划章节数、每章目标字数**，例如：目标 20000 字、10 章、每章不少于 1800 字/目标约 2000 字；章节数可 9–11 章）
  ↓
AI输出：改写策略（临时文档，改写完毕后删除，不持久保存）
  ↓
用户确认策略（可选）
  ↓
执行改写（使用输出模板文档）
  ↓
交付前自检：按「字数不计标点符号」统计正文（自「1」至最后一章末，不含小说标签/结语/创作手记）字数
  ↓
若正文 ≥ 8000 字 → 可交付（建议达到约 2 万字）；若 < 8000 字 → 扩写或重新改写，不得交稿（本规范中所有字数要求均不计标点符号）
```
**约定：** 改写策略仅作为本次改写的临时依据，可写在临时文档中；改写完成后删除该临时文档，不在流程目录下持久保留策略文件。**正文字数不足 8000 字不得交付，须扩写或重写直至达标；目标正文约 2 万字（1.8 万–2.2 万）。本规范中所有字数均不计标点符号。**

**输出与保存：** 改写完成后，成稿须保存到指定文件夹，详见《短篇小说输出模板》中的「输出与保存要求」。默认保存路径：`D:\软件\Obsidian项目\test\小说作品\短篇小说`，**文件名须带序号**，格式为 `NN-《[新标题]》.md`（如 `01-《逆光生长》.md`）。正文格式（含**对话与引用须使用英文双引号 `" "`**，不得使用直角引号「」）须符合该模板中的「正文输出约定」。

## 二、原小说分析规范

### 2.1 分析要素

**AI必须对原小说进行以下分析：**

```markdown
□ 基本信息
  - 原标题：[标题]
  - 字数：XXXX字

  - 风格：[甜宠/虐恋/轻松/等]
  - 核心情感：[爱情/复仇/成长/等]
  - 故事模式：[一言难尽/破镜重圆/欢喜冤家/等]

□ 核心情节
  - 开头：[如何开始]
  - 发展：[如何推进]
  - 高潮：[如何爆发]
  - 结局：[如何结束]
  - 核心冲突：[是什么]

□ 人物关系
  - 主角关系：[前任/夫妻/陌生人/等]

□ 情感基调

  - 可以优化的部分：[节奏/爽点/钩子]
```

## 三、差异化策略规范

### 3.1 差异化策略（含策略6：结构重组）

**AI必须从以下策略中选择至少2种（策略6 结构重组为强力推荐，可优先考虑）：**

#### 策略1：换人设

```markdown
原小说人设 → 改写后人设

示例：
原：女主是柔弱小白兔，男主是温柔暖男
改：女主是腹黑大佬型，男主是霸道总裁型

改写要点：
- 改变性格（但保留核心特质）
- 改变身份/职业
- 改变外貌特征
- 改变口头禅

保留：
- 核心关系（前任/夫妻/陌生人）
- 核心情感（爱恨交织/破镜重圆）
```

#### 策略2：换场景

```markdown
原场景 → 改写后场景

示例：
原：咖啡厅相遇、餐厅冲突
改：机场相遇、宴会冲突、公司冲突

改写要点：
- 换场景类型（从私密到公开，从和平到冲突）
- 换场景氛围（从温馨到紧张）
- 换时间（从白天到晚上，从工作日到周末）

保留：
- 场景功能（相遇/冲突/和解）
```

#### 策略3：换情节模式

```markdown
原情节模式 → 改写后情节模式

示例：
原：慢热升温型（慢慢相爱）
改：一见钟情型（快速相爱）
或：误会型（有误会导致冲突）
或：契约型（被迫在一起）

改写要点：
- 改变情节推进方式
- 改变冲突类型
- 改变和解方式

保留：
- 基本走向（相遇→相知→相爱）
```

#### 策略4：换表达方式

```markdown
原表达方式 → 改写后表达方式

示例：
原：文艺含蓄型
改：直白爽快型（番茄风格）

改写要点：
- 改变叙述方式（从第一人称到第三人称）
- 改变对话风格（从含蓄到直白）
- 改变描写方式（从细腻到简洁）
- 改变节奏（从慢到快）

保留：
- 核心事件（发生过的事）
```

#### 策略5：换结局方式

```markdown
原结局 → 改写后结局

示例：
原：开放式结局（留悬念）
改：圆满结局（明确在一起）
或：悲剧改成喜剧
或：改写成有番外的结局

改写要点：
- 改变结局走向
- 改变结局方式
- 改变结局氛围

保留：
- 情感基调（如果合适）
```

#### 策略6：结构重组（强力推荐）

```markdown
原结构：线性叙事（A → B → C → D）

改写后：倒叙/插叙开局（D → 回忆A → B → C → D高潮）

示例：
原书从校园恋爱写到婚变；改写后开篇就是离婚法庭（高潮前置），校园往事作为闪回穿插。

改写要点：
- 改变叙事顺序（高潮前置、关键冲突前置）
- 用闪回/插叙补全前因
- 保持时间线在读者脑中可还原

保留：
- 核心事件与情感走向

差异点：彻底改变了文本的指纹结构，安全性最高。
```

### 3.2 策略组合建议

```markdown
推荐组合：

组合1（最推荐）：
- 换人设 + 换场景 + 换表达方式
- 改动：70%
- 保留：30%（核心情感+基本框架）
- 差异度：最大

组合2（结构重组优先）：
- 结构重组 + 换人设 或 换场景
- 开篇即高潮/冲突，闪回补全前因
- 文本指纹改变最大，安全性最高

组合3：
- 换人设 + 换结局方式
- 改动：50%
- 保留：50%
- 差异度：中等

组合4：
- 换场景 + 换表达方式
- 改动：40%
- 保留：60%
- 差异度：较小
```

## 四、改写程度控制规范

### 4.1 人物名称与场景不得与对标重复（硬性规定）

**改写后的人物名称、场景均不得与对标短篇小说中出现的任何人物、场景相同或重复。**

```markdown
□ 人物名称
  - 改写文中出现的**所有人物名称**（主角、配角、提及的第三人等）**不得与对标短篇中任何人物名称相同或重复**。
  - 须全部替换为新的、与对标原文无关的人名，不得沿用或部分沿用对标中的人名。

□ 场景
  - 改写文中出现的**所有场景**（含具体地名、场所名、学校/公司/楼宇/店铺/城市名等）**不得与对标短篇中出现的任何场景、地名、场所名相同或重复**。
  - 须全部替换为新的、与对标原文无关的场景与名称，不得沿用或部分沿用对标中的场景名、地名。
```

**说明：** 即便对标原文只有一处出现某名，改写文中也**不得再使用**该人物名或该场景/地名。此条与「差异化」配合，降低与原文雷同风险。

### 4.2 保留与改写比例

```markdown
✅ 可以保留（30%）：
- 核心情感：爱情/复仇/成长的基本情感
- 基本框架：故事的基本走向
- 人物关系模式：前任/夫妻/陌生人的基本关系
- 核心事件：发生过的事件（但细节全改）

❌ 必须改写（70%）：
- 所有对话内容：100%重写
- 所有场景描写：100%重写
- 所有细节描写：100%重写
- 所有表达方式：100%重写
- **所有人名/地名/场景名：100%改，且不得与对标短篇中任何人物名、场景名重复**
- 所有心理描写：100%改

⚠️ 需要优化（根据情况）：
- 节奏：加快/放缓
- 爽点：增加/减少
- 钩子：增加/调整
- 情感浓度：加强/减弱
```

### 4.3 改写程度示例

**示例：咖啡厅相遇场景**

**原小说：**
```
林小雨坐在咖啡馆的角落里，手里捧着一杯热咖啡，望着窗外发呆。这是她和陈浩分手的第三天，心里还是很难受。她想起了三年前和陈浩在一起的点点滴滴，那时候的他们多甜蜜啊，可是现在一切都结束了。

"小雨，你怎么一个人在这里？"

林小雨抬头，看到是她的闺蜜周周。周周在她对面坐下，看着她红肿的眼睛，叹了口气。

"我……我只是想一个人静一静。"林小雨的声音有些沙哑。

"你不是为了陈浩吧？那个渣男，不值得你这样。"周周恨铁不成钢地说。

"可是我忘不了他……"林小雨的眼泪又流了下来。
```

**改写后（番茄风格）：**
```
"签了它，这一千万就是你的。"

林浅看着桌上的离婚协议。
笔尖，落在纸上。

刷刷刷。

签得飞快。

顾寒川愣住了。
"你……这么干脆？"

"怎么？"林浅抬眼，轻笑，"顾总舍不得？"

"……"

"那就这样吧。"
她把协议扔回去。
转身就走。

没有哭。
没有闹。
甚至，没有回头。

顾寒川看着她的背影。
（不对……她怎么走得这么干脆？）
```

**差异点：**
- ✅ 人名不同（林小雨→林浅，陈浩→顾寒川）
- ✅ 场景不同（咖啡馆→别墅）
- ✅ 冲突方式不同（分手回忆→直接离婚现场）
- ✅ 节奏更快（直接进入冲突）
- ✅ 有内心OS（番茄风格）
- ✅ 女主不软弱（当场反击）
- ✅ 对话100%重写
- ✅ 场景描写100%重写

## 五、番茄短篇小说风格要求

### 5.1 篇幅要求

**硬性要求：** 改写后**正文内容**（自章节「1」至最后一章末，不含小说标签、结语、创作手记）**不得少于 8000 字**。**字数均不计标点符号**（只计汉字、字母、数字等字符）。

- **交付前必须按「不计标点」统计正文字数**；若不足 8000 字，**不得交稿**，须扩写或重新改写直至达标。
- **目标正文篇幅：约 2 万字**（1.8 万–2.2 万，约 10 章、每章约 2000 字），便于一次改写达标；以上字数均不计标点符号。

```markdown
番茄短篇小说分类：
- 短篇：1-3万字（10章，每章1500-3000字）
- 中篇：3-5万字（10章，每章3000-5000字）

对于改写：目标约 2 万字（18,000–22,000 字）
- 对应章节数：约 10 章（每章约 2000 字）
- 可根据原文长度与策略微调，允许±10%浮动（1.8万–2.2万字）
```

### 5.1.1 章节与每章字数规定（必达）

**制定策略与改写时必须遵守以下规定：**

```markdown
□ 章节数
  - 以 **10 章** 为基准，可根据情节需要微调（建议 9–11 章）
  - 章节数不宜过多，以免每章过短、总字数难达标

□ 每章字数
  - **每章不少于 1800 字**（不计标点），**目标约 2000 字**
  - 单章建议不超过 2500 字，避免单章过长影响节奏

□ 执行要求
  - 每章写完后自检本章字数（不计标点）
  - **若某章不足 1800 字，须先在该章内扩写达标后再写下一章**
  - 全文写完后再按「不计标点」统计正文总字数，不足 8000 字不得交付，建议达到约 2 万字
```

### 5.2 番茄短篇小说特点

```markdown
与长篇的区别：
1. 节奏更快：没有时间慢热
2. 冲突更集中：所有冲突压缩在短篇幅内
3. 爽点更密集：每章都要有爽点
4. 钩子更强：章章都要有钩子
5. 结局更明确：不能含糊其辞
```

### 5.3 生动鲜活与爆款写作风格（必达）

**正文须生动鲜活，符合番茄小说平台爆款写作风格。改写时必须落实以下要求：**

```markdown
□ 生动鲜活（读者能「看见、听见、感受到」）
  - 有具体画面：场景、动作、神态用具体描写呈现，少用「很伤心」「很生气」等抽象概括
  - 对话有张力：对白有冲突、有潜台词、有情绪起伏，能推动情节或暴露人设
  - 细节有代入感：用一两处精准细节（小动作、物品、环境）代替大段说明
  - 情绪饱满：喜怒哀乐通过行为与对话外化，不让读者猜不到角色在想什么

□ 番茄爆款写作风格
  - 开篇抓人：第一章前几段内出现冲突、悬念或强情绪，不慢热
  - 节奏紧凑：少背景铺陈、少无关闲笔，每段推进情节或情绪
  - 每章有钩子：章末留悬念、反转或「下一幕」期待，让人想继续点
  - 爽点明确：打脸、逆袭、甜宠、解气等节点要写足、写透，不让读者觉得「就这？」
  - 语言有网感：短句、口语化、适当金句或内心 OS，避免文艺腔、翻译腔

□ 禁止
  - 空泛叙述（如「过了很久」「他们感情越来越好」而无具体场面）
  - 平淡流水账（按时间罗列事件，无冲突、无情绪高峰）
  - 大段说明性心理（「她心想……」「他觉得……」成段出现）
```

### 5.4 番茄短篇小说结构

```markdown
推荐结构：三段式

第一段：相遇+冲突（30%篇幅）
- 开篇即冲突
- 快速建立人物关系
- 留下钩子

第二段：发展+升温（40%篇幅）
- 情感推进
- 小爽点密集
- 中型冲突

第三段：高潮+结局（30%篇幅）
- 大型爽点
- 矛盾解决
- 圆满结局
```

### 5.5 视觉化与断句规范

```markdown
拒绝长难句：单句尽量不超过 15 个字。多拆成短句，便于扫读与情绪推进。

动作流描写：多用动词，少用形容词。
- 反例：「她很愤怒。」
- 正例：「她反手就是一巴掌。」

情绪外化：心理活动尽量通过微表情和肢体动作展现，减少大段内心独白。
- 反例：大段「他心想……」「她感到……」式内心戏。
- 正例：用眼神、小动作、语气变化、停顿等让读者自己推断情绪。

段落之间要有呼吸感：
- 段落与段落之间须用空行分隔，不得大段密排（连续多段无空行）。
- 适当运用短段、留白：关键句、情绪转折、重要对话后可单独成段或留空行，让阅读有节奏、有停顿。
- 避免整页冗长不分段，也避免每句都断成一段；长短段交替，视觉与节奏舒适。
```

## 八、使用提示

**给用户的提示：**
1. 输入短篇小说时，可以附加要求
2. AI会先制定改写策略
3. 确认策略后，再执行改写
4. 如果不满意，可以调整策略

**给AI的提示：**
1. 必须先分析原小说
2. 必须制定明确策略（策略可落于临时文档，改写完成后删除）；**策略中必须写明：目标正文字数约 2 万字（1.8万–2.2万，不得低于 8000）、计划章节数以 10 章为基准（可 9–11 章）、每章不少于 1800 字/目标约 2000 字**，避免写成「章节很多、每章很短」导致正文不足
3. 必须确保差异度
4. 策略必须可行
5. **改写时按「每章目标字数」落实**：动笔前心中有数（目标约 2 万字、章节数以 10 章为基准可 9–11 章、每章不少于 1800 字/目标约 2000 字，字数均不计标点），每写完一章自检本章字数，**若本章不足 1800 字须先在本章内扩写达标后再写下一章**；全文写完后再按「不计标点」统计正文总字数，不足 8000 字不得交付，建议达到约 2 万字
6. **人物名称与场景不得与对标重复**：改写正文中不得出现对标短篇小说里已有的任何人物名称、地名、场所名；制定策略时须列出对标中人名与场景并全部替换为新名，改写时不得沿用对标中的人名与场景。
7. 输出必须完整，且正文为可直接复制粘贴的纯正文（不包含作品信息、字数、章尾钩子等元数据标签；章节用数字 1、2、3… 表示，详见《短篇小说输出模板》）；**结语**放在正文标题后、正文前；正文结束后须接「小说标签」小节（3–8 个标签），再接创作手记
8. 改写完成后，将成稿保存到《短篇小说输出模板》规定的文件夹：`D:\软件\Obsidian项目\test\小说作品\短篇小说`，**文件名须带序号**：`NN-《[新标题]》.md`（如 `01-《逆光生长》.md`）
