# 短篇小说改写检查规范

> 版本：v1.5
> 创建日期：2026-02-15
> 用途：检查改写后的短篇小说质量
> 核心功能：差异度检查、质量检查、完整性检查

---

## 一、使用说明

### 1.1 文档用途
改写完成后，AI必须基于本规范进行**全面检查**，确保：
1. 与原小说的相似度<60%
2. **内容生动鲜活，符合番茄小说平台爆款写作风格**（有画面感、对话有张力、情绪饱满、每章有钩子有爽点；无空泛叙述、无流水账）
3. 符合番茄短篇小说风格
4. 质量达标，可以发布
5. **正文字数（一票否决）**：正文（自章节「1」至最后一章末，不含小说标签、结语、创作手记）**不少于 8000 字**；**目标约 2 万字**（1.8 万–2.2 万）。**字数均不计标点符号**。检查时先按「不计标点」统计正文字数；**若不足 8000 字，一律判定不通过**，须扩写或重新改写，不得交付；不足约 2 万字则建议扩写至目标范围。
6. **正文格式**符合《短篇小说输出模板》中的「正文输出约定」（含**生动鲜活、番茄爆款风格**、**段落之间有呼吸感**）：正文为可直接复制粘贴的纯正文，无【作品信息】、**字数：**、**【章尾钩子】**、**【全文完】**等标签，无 `---` 分隔符；章节标题为数字 1、2、3…，非「第X章：标题」；**对话与引用**一律使用英文双引号 `" "`，不得使用直角引号「」；**段落之间须有呼吸感**（段落间空行分隔，无大段密排，长短段交替）
7. **输出与保存**：成稿已保存到指定文件夹 `D:\软件\Obsidian项目\test\小说作品\短篇小说`，**文件名须带序号**，格式为 `NN-《[新标题]》.md`（如 `01-《逆光生长》.md`）
8. **结语与小说标签**：标题后、正文前须包含「结语」小节；正文之后须包含「小说标签」小节（3–8 个标签），不得遗漏

### 1.2 使用流程
```
改写完成
  ↓
AI执行：基于本规范全面检查
  ↓
AI输出：检查报告 + 修改建议
  ↓
用户确认/修改
  ↓
AI修改后重新检查
  ↓
输出最终版本
```

## 二、差异度检查规范

### 2.1 相似度计算方法

**AI必须计算改写后与原小说的相似度：**

```markdown
公式：
整体相似度 =
  对话相似度 × 30% +
  情节相似度 × 40% +
  场景相似度 × 30%

要求：< 60%
如果≥60%，必须继续改写！

一票否决：逻辑崩坏
- 若存在「逻辑崩坏」（见下方 2.3 逻辑自洽性检查），则无论相似度多少，整体视为不通过，必须重写问题段落。
- 改写最大风险之一：换了人设/背景，但配角仍按原人设对待主角，或文中残留与原设定冲突的细节。

「相似」的界定说明：
- 对话相似：句式或措辞高度雷同，可视为照搬或近义改写（换几个字但意思与语气基本一致即算相似）。
- 情节相似：指情节的**表述方式、描写与细节**是否雷同；与「是否保留同类故事节点」无关。允许保留「相遇→相知→相爱」等基本走向，但同一情节点的具体写法、对白、环境描写等必须重写，否则计为相似。
- 场景相似：场景类型相同且环境描写、氛围描写在措辞或结构上高度雷同。
```

#### 2.1.1 对话相似度检查

```markdown
「相似对话」界定：句式或措辞高度雷同、可视为照搬或近义改写。
- 算相似：原「你怎么在这里？」改写成「你怎么会在这儿？」（近义、句式一致）
- 不算相似：原「你怎么在这里？」改写成「顾总也来喝咖啡？」（意思、场合、语气均不同）

检查方法：
1. 提取改写后的所有对话
2. 提取原小说的所有对话
3. 逐一对比（按上述界定判断是否相似）
4. 计算：相似对话数 ÷ 总对话数

标准：
- 相似度 < 30%：✓ 通过
- 相似度 30%-50%：⚠️ 需要注意
- 相似度 ≥ 50%：❌ 不通过，必须改写

检查项：
□ 所有对话都重写了
□ 没有照搬原对话
□ 对话内容不同
□ 对话风格不同
□ 对话数量不同（增加/减少）
```

#### 2.1.2 情节相似度检查

```markdown
「情节相似度」界定：主要看情节的**呈现方式、描写与细节**是否重写，而非仅看故事节点是否相同。允许保留「核心事件」「基本框架」，但同一情节点的具体叙述、对白、心理与环境描写等若高度雷同，则计为相似。

「段落」界定：此处「段落」指**一个完整情节点或一个场景单元**（非自然段）。例如：「第一次见面」「第一次吵架」「误会解除」「表白」各算一个情节点；同一场景内不同小节可合并为一个场景单元。

检查方法：
1. 将原文与改写文按情节点/场景单元切分
2. 对比每个情节点内的表述方式、描写与细节（非仅对比「发生了什么事」）
3. 判断该情节点在改写中是否重写充分
4. 计算：相似情节点（或场景单元）数 ÷ 总情节点数

标准：
- 相似度 < 50%：✓ 通过
- 相似度 50%-70%：⚠️ 需要注意
- 相似度 ≥ 70%：❌ 不通过，必须改写

检查项：
□ 情节走向或节点可有保留，但表述方式不同
□ 情节细节与描写已重写
□ 情节数量不同（增加/减少/删减）或顺序有调整
```

#### 2.1.3 场景相似度检查

```markdown
检查方法：
1. 对比场景类型
2. 对比场景描写
3. 对比环境氛围
4. 计算：相似场景 ÷ 总场景

标准：
- 相似度 < 40%：✓ 通过
- 相似度 40%-60%：⚠️ 需要注意
- 相似度 ≥ 60%：❌ 不通过，必须改写

检查项：
□ 所有场景都重写了
□ 场景类型不同
□ 场景描写100%重写
□ 场景数量不同
□ 场景顺序不同
```

### 2.2 差异化检查清单

**AI必须逐项检查：**

```markdown
□ 人物名称与场景不得与对标重复（一票否决）
  - [ ] 改写文中**未出现**对标短篇小说中的任何人物名称（主角、配角、提及的第三人等）
  - [ ] 改写文中**未出现**对标短篇小说中的任何场景名、地名、场所名（学校/公司/楼宇/店铺/城市等）
  - 若有任一处与对标重复，一律判定不通过，须替换后重检

□ 人名差异（必须100%不同）
  - [ ] 女主名字改了
  - [ ] 男主名字改了
  - [ ] 配角名字改了
  - [ ] 所有名字都改了

□ 地名/公司名差异（必须100%不同）
  - [ ] 地名都改了
  - [ ] 公司名都改了
  - [ ] 所有名称都改了

□ 对话差异（必须100%重写）
  - [ ] 所有对话都重写了
  - [ ] 没有照搬原对话
  - [ ] 对话风格符合番茄

□ 场景差异（必须100%重写）
  - [ ] 所有场景都重写了
  - [ ] 场景描写100%重写
  - [ ] 或场景类型都换了
  - [ ] 场景名/地名与对标无任何重复

□ 情节差异（必须重写）
  - [ ] 情节细节都重写了
  - [ ] 或情节模式换了
  - [ ] 情节顺序调整了

□ 表达差异（必须100%重写）
  - [ ] 所有表达都重写了
  - [ ] 叙述方式改了
  - [ ] 语言风格改了

□ 人设差异（必须有差异）
  - [ ] 女主人设改了
  - [ ] 男主人设改了
  - [ ] 人设差异明显
```

### 2.3 逻辑自洽性检查（一票否决项）

**存在以下任一项即视为逻辑崩坏或违规，整体不通过，须重写该段落或相关设定：**

```markdown
□ 人物名称或场景与对标重复
  - 若改写文中出现对标短篇小说中的任何人物名称、地名、场所名，视为违规，一律不通过，须全部替换后重检。

□ 人设与行为是否冲突
  - 例：改写后女主是「腹黑大佬」，却仍出现「被女配欺负不敢说话」等原剧情节，即为人设与行为冲突。
  - 检查：配角是否仍按「原人设」对待主角？主角的应对是否符合「新人设」？若有违和，视为逻辑崩坏，必须重写该段落。

□ 背景与道具是否冲突
  - 例：场景从现代换到古代，却误留「手机、电梯、微信」等词汇或设定。
  - 检查：时代、世界观、职业设定是否统一？有无漏改的现代/原场景专属词汇或道具？若有，视为逻辑崩坏，必须修正。
```

## 三、质量检查规范

### 3.1 番茄风格检查

```markdown
□ 生动鲜活与爆款风格（必检）
  - [ ] 内容生动鲜活：有具体画面、对话有张力、细节有代入感、情绪饱满可感
  - [ ] 符合番茄小说平台爆款写作风格：开篇抓人、节奏紧凑、每章有钩子有爽点、语言有网感
  - [ ] 无空泛叙述、无平淡流水账、无大段说明性心理

□ 节奏检查
  - [ ] 节奏快，不拖沓
  - [ ] 开头即冲突（如果是短篇小说）
  - [ ] 中间不拖沓
  - [ ] 结局干脆利落

□ 语言检查
  - [ ] 语言直白不文青
  - [ ] 短句为主
  - [ ] 口语化
  - [ ] 有网感

□ 情绪检查
  - [ ] 情绪浓烈
  - [ ] 爱恨分明
  - [ ] 有爆发点
  - [ ] 读者能感受到

□ 爽点检查
  - [ ] 有爽点
  - [ ] 爽点明确
  - [ ] 爽感强烈
  - [ ] 读者会爽

□ 钩子检查
  - [ ] 每章有钩子（如果是短篇小说）
  - [ ] 钩子够强
  - [ ] 钩子连贯
  - [ ] 让人想追更
```

### 3.2 短篇小说专项检查

```markdown
□ 正文格式检查（符合《短篇小说输出模板》正文输出约定）
  - [ ] 正文为可直接复制粘贴的纯正文，无【作品信息】、**字数：**、**【章尾钩子】**、**【全文完】**等元数据标签
  - [ ] 章节标题为数字 1、2、3…，非「第X章：标题」
  - [ ] 对话与引用使用英文双引号 `" "`，未使用直角引号「」
  - [ ] **段落之间有呼吸感**（段落间空行分隔，无大段密排，长短段交替，节奏舒适）
  - [ ] 标题后、正文前含「结语」小节；正文之后含「小说标签」小节（3–8 个标签）

□ 篇幅检查
  - [ ] **已统计正文字数**（自「1」至最后一章末，不含小说标签、结语、创作手记；**字数不计标点符号**）
  - [ ] **正文 ≥ 8000 字**（不足则一票否决，须扩写或重新改写，不得交付）；**建议达到约 2 万字**（1.8 万–2.2 万）
  - [ ] 字数在目标范围（改写默认目标：约 2 万字，允许 1.8万–2.2万；均不计标点）
  - [ ] 章节数合理（以 10 章为基准，可 9–11 章；避免章节过多导致每章过短）
  - [ ] 每章字数达标（**每章不少于 1800 字**，目标约 2000 字，不计标点；若有章不足 1800 字且未扩写则不符合规范）

□ 结构检查
  - [ ] 有开头（冲突）
  - [ ] 有发展（升温）
  - [ ] 有高潮（爆发）
  - [ ] 有结局（圆满）

□ 完整性检查
  - [ ] 故事完整
  - [ ] 有头有尾
  - [ ] 情感连贯
  - [ ] 逻辑合理

□ 读者体验检查
  - [ ] 能吸引人
  - [ ] 能看下去
  - [ ] 有追读欲望
  - [ ] 有爽感
  - [ ] 读起来生动鲜活、有代入感（非流水账）
```
