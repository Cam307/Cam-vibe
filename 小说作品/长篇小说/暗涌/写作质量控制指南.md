# 《他订婚那天，我头也不回地走了》写作质量控制指南

> 本文档用于确保长篇连载中的逻辑一致性和质量稳定性

---

## 🛡️ 核心原则

**唯一真相源原则：** 所有时间线、人物设定、情节细节以 `人物档案.md` 和 `大纲设定/他订婚那天我走了-超详细大纲（30万字，含上下文约束）.md` 为准，任何矛盾以这两个文件为准。

---

## 📅 一、时间线管理（最易出错！）

### 1.1 核心时间线（绝不能错）

| 时间节点 | 年龄 | 事件 |
|----------|------|------|
| 18岁 | 大一 | 林深与陆行舟相识 |
| 22岁 | 大四毕业 | 林深进入陆氏集团 |
| 25岁 | 故事开始 | 第1章开始，相识**7年** |
| 25岁半 | 订婚宴 | 林深心碎离开 |
| 25-28岁 | 国外三年 | 林深在巴黎苦难 |
| 28岁 | 重逢 | 故事结局 |

### 1.2 时间线速查表

**写作时随时对照：**

```
当前年龄 = 开始年龄(25) + 经过年数
相识年数 = 当前年龄 - 18
```

**错误示例：**
- ❌ "十年了" （正确：7年）
- ❌ "从小学就认识" （正确：大一认识）
- ❌ "工作了五年" （正确：工作3年，22-25岁）

### 1.3 每章写作前检查

在写新章节前，先确认：

- [ ] 本章发生在什么时间点？
- [ ] 距离上一章过了多久？
- [ ] 人物当前年龄是多少？
- [ ] 相识时间是几年？
- [ ] 是否有特殊时间节点（生日、纪念日）？

### 1.4 敏感词避雷

**这些词需要仔细核对时间：**

| 词汇 | 使用前必问 |
|------|-----------|
| "十年/二十年/一辈子" | 具体几年？算了没？ |
| "从小/童年" | 从几岁开始？ |
| "大学时期" | 大几？ |
| "工作了X年" | 毕业几年了？ |
| "认识X年了" | 当前年龄-18=？ |

## 👤 二、人物设定一致性

### 2.1 核心设定速查

| 人物 | 核心特征 | 绝对不能写 |
|------|----------|-----------|
| 林深 | 清冷美人、温和→冷淡、设计师 | 不要写他"粗鲁、暴躁" |
| 陆行舟 | 傲慢→偏执深情、商业精英 | 不要写他"懦弱、无能" |
| 奶奶 | 已故（林深在国外时去世） | 不要写奶奶"还活着、通电话" |

### 2.2 每章人物检查

**林深行为对照：**
- [ ] 符合当前性格阶段？（前期温和/后期冷淡）
- [ ] 对陆行舟的态度是否合理？（第1-20章：温柔隐忍；第26章后：冷淡疏离）
- [ ] 外貌描写是否一致？（178cm、清冷美人）

**陆行舟行为对照：**
- [ ] 符合当前性格阶段？（前期傲慢/后期偏执追妻）
- [ ] 对林深的感情是否合理？（第1-25章：逃避；第26章后：疯狂追妻）
- [ ] 外貌描写是否一致？（188cm、俊美锋利）

### 2.3 人物关系检查

**写作前确认当前关系阶段：**

| 章节范围 | 关系状态 | 关键词 |
|----------|----------|--------|
| 1-20章 | 暧昧期（双向暗恋未说破） | 最好的朋友、默契、不敢说 |
| 21-25章 | 破裂期（订婚→离开） | 心碎、订婚宴、离开 |
| 26-45章 | 重逢期（冷淡疏离） | 陆总、林设计师、心墙 |
| 46-65章 | 追妻期（攻疯狂追妻） | 给我机会、我知道错了 |
| 66-80章 | 修复期（慢慢HE） | 慢慢来、原谅 |

## 🔍 三、情节逻辑检查

### 3.1 伏笔回收记录

| 伏笔 | 设置章节 | 回收章节 | 状态 |
|------|----------|----------|------|
| 苏婉回国 | 第1章 | ？ | 待回收 |
| 陆行舟不敢面对感情 | 第1章 | ？ | 待回收 |
| 林深奶奶身体不好 | 第1章 | 第21-25章 | 待回收 |
| 林深设计才华 | 第1章 | 全书 | 进行中 |

### 3.2 矛盾点检查清单

每写完一章，检查：

- [ ] 本章情节是否与前面章节矛盾？
- [ ] 人物行为是否符合其性格？
- [ ] 时间线是否对得上？
- [ ] 是否有未解释的突然转变？
- [ ] 前文提到的人/事是否处理了？

### 3.3 常见逻辑陷阱

**避坑指南：**

| 陷阱类型 | 示例 | 如何避免 |
|----------|------|----------|
| 时间矛盾 | "十年了" vs "七年了" | 写作用时间线速查表 |
| 人设崩塌 | 林深突然暴躁 | 对照人物设定检查 |
| 突然遗忘 | 前文说奶奶去世，后文打电话 | 记录伏笔，定期回顾 |
| 地点错误 | A市变B市 | 建立地点记录表 |
| 年龄混乱 | 25岁写成26岁 | 每章开头标注时间 |

## 📝 四、章节写作流程

### 推荐工作流

```
写作前 → 写作中 → 写作后 → 发布前
   ↓        ↓        ↓        ↓
查设定   对时间   查逻辑   最终审
定时间   查人设   找矛盾   看全局
```

### 步骤1：写作前（5分钟）

1. **确认时间点**
   - 当前章节时间：年/月/日
   - 距离上一章：过了多久
   - 人物当前年龄

2. **确认关系阶段**
   - 参考上表"人物关系检查"

3. **回顾相关设定**
   - 打开 `人物档案.md` 确认人物特征
   - 打开大纲确认本章节在故事中的位置

### 步骤2：写作中（随时）

1. **涉及时间时** → 停下来，算一算
2. **涉及人物特征时** → 查人物设定
3. **涉及前文情节时** → 搜索关键词确认

### 步骤3：写作后（10分钟）

1. **运行检查清单**
   ```bash
   # 搜索敏感词
   grep -n "十年\|二十年\|从小" 章节文件.md
   ```

2. **对照检查表**
   - [ ] 时间线正确
   - [ ] 人物行为合理
   - [ ] 无明显矛盾

### 步骤4：发布前（建议）

- [ ] 放置1小时再读一遍
- [ ] 从读者角度检查是否理解
- [ ] 检查错别字和标点

## 🧪 五、自检工具

### 5.1 敏感词搜索脚本

写完章节后，运行以下命令检查常见错误：

```bash
# 搜索时间相关敏感词
grep -n "十年\|二十年\|从小\|童年" 第X章-xxx.md

# 搜索可能的人设矛盾
grep -n "暴躁\|粗鲁\|懦弱" 第X章-xxx.md

# 检查已故角色是否被误用
grep -n "奶奶.*电话\|奶奶.*说" 第X章-xxx.md
```

### 5.2 时间线快速计算

```
当前相识年数 = 当前年龄 - 18

示例：
- 25岁时：25 - 18 = 7年 ✓
- 28岁时：28 - 18 = 10年 ✓
```

### 5.3 章节信息模板

每章开头建议添加（注释）：

```markdown
<!--
时间：故事开始后第X天/月/年
季节：秋
林深年龄：25岁
陆行舟年龄：25岁
相识时间：7年
关系阶段：暧昧期
-->
```

## 📋 六、常见问题速查

| 问题 | 检查项 | 解决方法 |
|------|--------|----------|
| 时间线乱了 | 搜索"X年"类词汇 | 对照核心时间线表修正 |
| 人设崩了 | 检查人物行为 | 查人物设定档案 |
| 情节矛盾 | 回顾前文 | 搜索相关情节关键词 |
| 忘了伏笔 | 查伏笔记录表 | 标注"待回收"定期回顾 |
| 逻辑不通 | 问"为什么合理" | 为关键转折提供铺垫 |

```
□ 我知道这一章发生在什么时间
□ 我知道人物当前年龄是多少
□ 我知道两人认识了几年
□ 我知道当前关系处于哪个阶段
□ 我知道本章要推动什么情节
```

## 🎯 八、终极建议

1. **建立权威** - `人物档案.md` 和 `大纲设定/他订婚那天我走了-超详细大纲（30万字，含上下文约束）.md` 是唯一真相源，任何矛盾以它们为准

2. **勤查勤记** - 不确定的马上去查，不要凭印象

3. **工具辅助** - 用grep搜索关键词，不要靠肉眼

4. **定期回顾** - 每10章回顾一次，检查整体逻辑

5. **读者视角** - 写完后从读者角度读一遍，发现问题
